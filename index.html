Introduction

WoodCore is a lightweight cloud banking platform that enables fintech, digital banks, and payment service providers to create their own products requiring banking capabilities, or to integrate such capabilities into their systems, apps. and workflows.

Customers connect to the platform via the Customer API and access financial products - a full list of the capabilities available via the WoodCore application can be seen below or on ours.

The platform is driven by a simple API, built to be rapidly integrated into your business. WoodCore is all about simplifying your life by allowing you to integrate a complete set of financial services capabilities into your business, app. or product.

These guides are designed to help anyone and everyone use the API - from CEOs thru to developers. However, if you need more detailed, development-focused information after going through the guides, give us a quick heads-up on tech@woodcoreapp.com

Pick the docs that are right for you. These guides, and API reference docs will get you across the deploy line, straight to HTTP 200 OK.

Authorization

Authentication to the API is configured to be supported via HTTP Auth.

We provide secret keys. The secret keys however, are to be kept secret. If for any reason you believe your secret key has been compromised or you wish to reset them, you can do so by speaking to a CSM or sending an email to tech@woodcoreapp.com.


Authorization headers

Header Value Description
WoodCore-Tenant tenant_name tenant_name will enable woodCore to supply specific resources
Authentication base64 key base64 authentication key
apiKey encrypted key hash for protecting your resources

API Errors

WoodCore API is RESTful and as such, uses conventional HTTP response codes to indicate the success or failure of requests.
Common HTTP response codes

Status Description
200 OK - Everything Worked.
400 Bad Request - Invalid Parameter or Data Integrity Issue.
401 Authentication Error.
403 Unauthorized Request.
404 Resource Not Found
500 Platform Internal Server Error.

Customers or Clients can be created with a deposit/savings accounts alongside. This API exists to enable client activation (for when a client becomes an approved member of the financial Institution). if the active is set to true, the customer is activiated at the time of creation.

If the client happens to be already active this API will result in an error.

HTTP Request

POST https://api.woodcoreapp.com/v1/clients

Query Parameters

Parameter Required Description
officeId true branch id
firstname true customers first name
lastname true customer last name
externalId false optional unique identifier
dateFormat true standard date format
locale true language
active true confirm if the customer should be active or not
activationDate true date of activation
submittedOnDate true submittion date
savingsProductId false savings product id
